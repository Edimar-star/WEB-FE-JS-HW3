JavaScript Async Concepts Explained

1. JavaScript Async

JavaScript Async is the mechanism that allows operations to be performed in the background, without blocking the main thread.


console.log("Start");

setTimeout(() => {
    console.log("This is async");
}, 2000);

console.log("End");

2. Callback

A callback is a function passed into another function as an argument, which is invoked after the main function has finished executing.


function greeting(name) {
    console.log('Hello ' + name);
}

function processUserInput(callback) {
    const name = 'John';
    callback(name);
}

processUserInput(greeting);

3. Promise

A promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value.


let promise = new Promise((resolve, reject) => {
    let success = true;
    if (success) {
        resolve("Operation successful");
    } else {
        reject("Operation failed");
    }
});

promise.then((message) => {
    console.log(message);
}).catch((message) => {
    console.log(message);
});

4. Async/Await

Async/Await is syntactic sugar built on top of promises, allowing you to write asynchronous code in a synchronous-looking style.


async function fetchData() {
    let response = await fetch('https://api.example.com/data');
    let data = await response.json();
    console.log(data);
}

fetchData();

5. Event Loop

The event loop is a mechanism that allows JavaScript to perform non-blocking I/O operations, despite being single-threaded, by offloading operations to the system's background (e.g., async APIs).


console.log("Start");

setTimeout(() => {
    console.log("Callback");
}, 0);

console.log("End");
